Checklist de Accesibilidad WCAG

Criterios de accesibilidad aplicados en el rediseño del sitio (segundo bimestre):

#Criterio WCAGEstado
1 Contraste de color — Texto vs fondo con ratio mínimo 4.5:1 (teal #00D2BE sobre negro #07070A cumple WCAG AA) Cumplido
2 Tamaño de tipografía — Mínimo 16px en cuerpo de texto, 12px en captions, escala proporcional Cumplido
3 Textos alternativos — Atributo alt en todas las imágenes; aria-label en SVGs Cumplido
4 Skip link — Enlace "Saltar al contenido principal" visible al recibir foco en todas las páginas Cumplido
5 Focus visible — Todos los elementos interactivos (botones, links, inputs) tienen outline visible al recibir foco con teclado Cumplido
6 Semántica HTML5 — Uso correcto de header, nav, main, section, article, footer Cumplido
7 Atributo lang — lang="es" en todas las páginas para lectores de pantalla Cumplido
8 ARIA labels — aria-label y aria-current en navbar; aria-hidden en elementos decorativos Cumplido

Decisiones de diseño

  1. Contraste teal sobre negro: El color #00D2BE sobre #07070A supera el ratio 4.5:1 requerido por WCAG AA para texto normal, garantizando legibilidad en todos los dispositivos.
  2. Jerarquía tipográfica: League Spartan 900 para títulos (grande, condensada, alto impacto) y Montserrat 300 para cuerpo (ligera, alta legibilidad) crean una jerarquía visual clara sin necesidad de colores adicionales.
  3. Navegación responsive: El navbar de Bootstrap colapsa automáticamente en móvil con un botón de menú accesible desde teclado, con aria-expanded y aria-controls correctamente configurados.
  4. Focus outline visible: Se sobreescribió el estilo de foco predeterminado del navegador para asegurar un outline teal de 2px visible en todos los elementos interactivos, crítico para usuarios de teclado.
  5. Skip link: El enlace "Saltar al contenido principal" es invisible visualmente pero accesible desde teclado, permitiendo a usuarios de lectores de pantalla saltar directamente al contenido.

Ventajas encontradas

  • Grid de 12 columnas: Organización responsive sin escribir media queries propias — col-sm, col-md, col-lg automatizan el layout.
  • Componentes listos: Navbar, accordion, modal, tooltip, badges y progress bars completamente funcionales con solo clases CSS y atributos data-bs.
  • Accesibilidad incorporada: Los componentes de Bootstrap incluyen aria-expanded, aria-controls y roles ARIA por defecto.
  • CDN sin instalación: Funciona desde un enlace externo, compatible con el flujo de trabajo actual sin Node.js ni compiladores.
