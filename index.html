¿Qué ocurre al ingresar una URL?

Cada dirección que escribes desencadena un proceso fascinante entre tu navegador y servidores remotos en milisegundos.

Ver el proceso completo

¿Qué es una URL?

Una URL (Universal Resource Locator) es la dirección única que identifica un recurso en la Web. El navegador interpreta cada segmento para saber a quién contactar y qué solicitar al servidor remoto.

Dato clave: Todo el proceso desde que escribes la URL hasta que ves la página ocurre en menos de 3 segundos.

Tipos de recursos web

Página HTML

El recurso más común. El servidor devuelve un documento de hipertexto que el navegador renderiza visualmente.

text/html

Imagen / Archivo

La URL apunta a un archivo binario como imagen JPG, PDF u otro recurso descargable desde el servidor.

image/jpeg

API / JSON

Las URLs también apuntan a servicios que devuelven datos estructurados en formato JSON o XML.

application/json

HTTPS Cifrado

Con https:// la comunicación viaja cifrada mediante TLS para proteger datos sensibles.

puerto :443

Glosario de términos web

  • URL — Uniform Resource Locator. Dirección que identifica un recurso en la Web.
  • HTTP — HyperText Transfer Protocol. Protocolo de comunicación cliente-servidor.
  • HTTPS — HTTP Secure. Versión cifrada con TLS/SSL del protocolo HTTP.
  • DNS — Domain Name System. Traduce nombres de dominio a direcciones IP.
  • TCP/IP — Protocolo de red que garantiza la entrega de datos entre máquinas.
  • DOM — Document Object Model. Árbol de nodos HTML construido por el navegador.
  • Renderizado — Proceso de convertir el HTML recibido en una página visual.
  • Servidor web — Software que escucha solicitudes HTTP y devuelve respuestas.
