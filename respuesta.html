Respuesta y Renderizado HTML5

El navegador recibe el HTML y lo convierte en la página visual que el usuario ve, interpretando las etiquetas semánticas.

Pipeline de renderizado

Al recibir el HTML el navegador construye el DOM, aplica CSS para crear el CSSOM, los combina en un árbol de renderizado y pinta los píxeles en pantalla.

Etiquetas semánticas HTML5

Etiqueta Propósito semántico Equivalente anterior Uso típico
<header>Encabezado de página o sección<div id="header">Logo, título, búsqueda
<nav>Navegación principal<div id="nav">Menú de enlaces
<main>Contenido principal único<div id="main">Artículo, formulario
<section>Agrupación temática<div class="section">Bloque temático
<article>Contenido independiente<div class="post">Entrada de blog
<aside>Contenido complementario<div id="sidebar">Barra lateral
<footer>Pie de página<div id="footer">Copyright, contacto
<figure>Contenido ilustrativo<div class="img">Imagen con caption

Beneficios del HTML semántico

  • Accesibilidad: Lectores de pantalla interpretan la estructura correctamente.
  • SEO: Los buscadores comprenden mejor el contenido.
  • Mantenibilidad: Código más legible y fácil de modificar.
  • Estándar W3C: Cumple las guías del World Wide Web Consortium.

Códigos de estado HTTP

CódigoNombreSignificado
200OKSolicitud exitosa, se devuelve el recurso.
201CreatedRecurso creado exitosamente (respuesta POST).
301Moved PermanentlyEl recurso se movió a una nueva URL.
304Not ModifiedUsa la versión en caché del navegador.
404Not FoundEl recurso solicitado no existe.
500Internal Server ErrorError interno del servidor.

Estructura HTML5 semántica correcta:

<!-- Código fuente de respuesta.html -->
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Renderizado HTML5</title>
</head>
<body>
  <header><h1>Título</h1></header>
  <nav><ul>...</ul></nav>
  <main>
    <section>
      <h2>Sección</h2>
      <p>Contenido.</p>
    </section>
  </main>
  <footer><p>&copy; 2026</p></footer>
</body></html>
