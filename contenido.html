El proceso completo de la Web

Desde la solicitud HTTP hasta el renderizado en el navegador — todo explicado paso a paso con diagramas y código real.

Los pasos del proceso web

HTTP (HyperText Transfer Protocol) regula la comunicación entre el cliente y el servidor con un modelo simple de petición–respuesta.

MétodoDescripciónUso
GETSolicita información del servidorNavegar páginas
POSTEnvía datos al servidorFormularios
PUTReemplaza un recurso completoActualizar datos
DELETEElimina un recursoBorrar registros

Ejemplo real de petición GET:

GET /tecnologias-web HTTP/1.1
Host:            www.utpl.edu.ec
User-Agent:      Mozilla/5.0 (Chrome/120)
Accept:          text/html
Accept-Language: es-EC, es;q=0.9
Connection:      keep-alive
(sin cuerpo en GET)
Documentación oficial: MDN Web Docs — Métodos HTTP

Un servidor web escucha solicitudes HTTP en el puerto 80 o 443. Al recibirlas, las procesa y devuelve la respuesta adecuada al cliente.

Estático

Sirve archivos guardados: HTML, CSS, imágenes, JS.

Dinámico

Ejecuta PHP/Python y genera HTML en tiempo real.

Cabecera HTTP de respuesta:

HTTP/1.1 200 OK
Server:         Apache/2.4.57
Content-Type:   text/html; charset=UTF-8
Content-Length: 4823
Cache-Control:  max-age=3600, public
(línea en blanco + HTML)
CódigoSignificado
200Solicitud exitosa
301Redirección permanente
404Recurso no encontrado
500Error interno del servidor

Al recibir el HTML el navegador construye el DOM, aplica CSS para crear el CSSOM, los combina en un árbol de renderizado y pinta los píxeles en pantalla.

Etiquetas semánticas HTML5

EtiquetaPropósito semánticoEjemplo de uso
<header>Encabezado de página o secciónLogo, título, búsqueda
<nav>Navegación principalMenú de enlaces
<main>Contenido principal únicoArtículo, formulario
<section>Agrupación temáticaBloque "Acerca de"
<article>Contenido independienteEntrada de blog
<aside>Contenido complementarioBarra lateral
<footer>Pie de páginaCopyright, contacto
<figure>Contenido ilustrativoImagen con caption

Estructura semántica de una página HTML5 correctamente organizada:

<!-- Estructura HTML5 semántica -->
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi Página</title>
</head>
<body>
  <header><h1>Título</h1></header>
  <nav><ul><li><a href="#">Inicio</a></li></ul></nav>
  <main>
    <section>
      <h2>Sección</h2>
      <p>Contenido.</p>
    </section>
  </main>
  <footer><p>&copy; 2026</p></footer>
</body></html>
Beneficios del HTML semántico:
  • Mejora la accesibilidad para lectores de pantalla
  • Favorece el SEO en motores de búsqueda
  • Código más legible y mantenible
  • Cumple los estándares W3C

HTTP vs HTTPS

CaracterísticaHTTPHTTPS
Puerto predeterminado80443
CifradoNoSí (TLS/SSL)

Certificado SSLNo requeridoRequerido
SEO GooglePenalizadoFavorecido

¿Por qué HTTPS?

HTTPS cifra toda la comunicación entre el navegador y el servidor mediante TLS/SSL, protegiendo contraseñas, datos personales y transacciones financieras.
