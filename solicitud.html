La Solicitud HTTP

El navegador construye y envía un mensaje estructurado al servidor siguiendo las reglas del protocolo HTTP.

¿Qué es el protocolo HTTP?

HTTP (HyperText Transfer Protocol) regula cómo se comunican el cliente (navegador) y el servidor. Funciona con un modelo de petición–respuesta: el cliente pide, el servidor responde.

Cada solicitud contiene un método (verbo), la URL del recurso, cabeceras con metadatos, y opcionalmente un cuerpo con datos.

Versiones: HTTP/1.1 (texto plano), HTTP/2 (binario, multiplexado), HTTP/3 (sobre QUIC/UDP).

Método Descripción Cuándo se usa Datos en

GET vs POST en detalle

GET Obtener datos

Los datos viajan en la URL. Es visible, cacheable y puede ser marcado como favorito. No se usa para contraseñas.

GET /buscar?q=html&lang=es HTTP/1.1
Host: www.utpl.edu.ec

POST Enviar datos

Los datos van en el cuerpo de la solicitud. Más seguro para información confidencial como contraseñas.

POST /login HTTP/1.1
Host: www.utpl.edu.ec
(cuerpo)
user=ana&pass=*****

Cabeceras HTTP más comunes

  • Host — Indica el dominio del servidor al que se dirige la petición.
  • User-Agent — Identifica el navegador y sistema operativo del cliente.
  • Accept — Especifica los tipos de contenido que acepta el cliente.
  • Content-Type — Indica el formato de los datos en el cuerpo.
  • Authorization — Contiene credenciales de autenticación.
  • Cookie — Envía cookies para mantener sesiones activas.
